Property Record

Format

Property records occur at the beginning of the file and contain information about the grammar as well as attributes that affect how the grammar functions. The record is preceded by a byte field that contains the value 112, the ASCII code for the letter 'p'. The record contains an index, the property name, and its associated value. The idea is to allow additional information to be added in the future. This may include more information about the grammar and/or user-defined meta-data.

Fields

Contents

Name Type Description
Index Integer The index value, described below, is unique for each different property stored in the file. Some properties have a specific index. This index, which is zero, delimited, can be stored directly in an array. Or, it can be ignored and the table can be constructed using the property's name.
Name String The name of the property.
Value String The value of the property.

Index Constants

Index Property Name Description
0 Name The name of the grammar.
1 Version This field contains the version of the grammar. The value can be any string; not just a number.
2 Author This is not a necessary field, but credit to the author should be included in any completed work.
3 About This is a general-purpose field the designer can use to specify some notes about the grammar. Like, the name, version and author, this field does not affect the actual parsing algorithms.
4 Character Set This field contains the character set used by the Character Set table. For now, it will always be 'Unicode'. However, other character sets could be supported in the future.
5 Character Mapping Any character mapping that is used is stored in this field.
7 Generated By This field contains the name and version of the application that created the file.
6 Generated Date Date the file was created in "yyyy-mm-dd hh:mm:ss" format.
